# Author docs/troubleshooting/launcher-gh-auth.md body

## Summary
Create the new troubleshooting doc [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) (creating the [docs/troubleshooting/](docs/troubleshooting/) directory if absent) that documents the launcher's gh-identity probe: when it runs, how to decode its diagnostic line, the two failure modes it prevents, and a verbatim symptoms→remediation table for each probe error path. This is a single focused authoring task that fits well within one PR group.

## Problem / Motivation
The launcher's gh-identity probe (in [scripts/Resolve-GhIdentity.ps1](scripts/Resolve-GhIdentity.ps1), wired in [scripts/Invoke-PolyphonySdlc.ps1](scripts/Invoke-PolyphonySdlc.ps1)) emits a diagnostic line and several specific error strings, but operators currently have no documentation explaining what the probe does, why it exists (competing-worker `gh auth switch` slippage; stale/wrong-scope token), or how to remediate each failure path. Without this doc, the EMU-vs-PolyphonyRequiem account-switch case in particular is opaque.

## Proposed Approach
Author a single new markdown file at [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) containing:

1. **Probe behavior section** — when it runs (`-Platform github` AND not `-DryRun`), and a decoding of the `[polyphony-sdlc] gh identity pinned: user='X' source=Y token_len=N` diagnostic line (what `user` is, `source` ∈ {`gh-keyring`, `env`}, what `token_len` means).
2. **Failure modes section** — name and explain both failure modes the probe prevents: (1) competing-worker `gh auth switch` slippage mid-run, (2) stale / wrong-scope token. Lift the inline prose from [scripts/Invoke-PolyphonySdlc.ps1](scripts/Invoke-PolyphonySdlc.ps1) (around lines 238–277).
3. **Symptoms → remediation table** — one row per probe error path, with the error string quoted **verbatim** from [scripts/Resolve-GhIdentity.ps1](scripts/Resolve-GhIdentity.ps1) and a concrete remediation command:
   - "gh not on PATH" → install/PATH-fix command
   - "no token cached" → `gh auth login`, plus the EMU-vs-PolyphonyRequiem account-switch case → `gh auth switch --user PolyphonyRequiem`
   - "token validation failed" (401/403) → re-login / scope-fix command
   - "gh auth token timed out" → retry guidance

Cross-links from other docs into this file are handled in sibling `issue-2` (out of scope here). Before authoring, open both source scripts and copy error strings byte-for-byte to ensure verbatim fidelity.
