## Summary
Tighten the kept-archive (sibling research repo populated by the archivist from Issue 4 of Epic #3107) into a discoverable Obsidian-friendly wiki. Add a Johnny-Decimal numbering scheme, Obsidian frontmatter / wiki-link conventions, an `INDEX.md` maintained atomically with each keep/expand, and — as Stretch — a citation freshness signal. Most items here are Stretch on the parent Epic; deliver the Minimum first and document any deferred Stretch.

## Problem / Motivation
The parent Epic [#3107](plans/plan-3107.md) introduces a research-augmented planning cycle whose archivist curation pass writes kept articles into a sibling research repo (see [plans/plan-3107.md](plans/plan-3107.md) lines 33–34). Without numbering, link conventions, and a topical index, that repo will degrade into an unbrowsable pile within a few runs: humans cannot navigate it in Obsidian, and downstream architect runs cannot reliably discover prior art. This Issue establishes the conventions and the index-maintenance protocol so the archive is useful from the first kept article.

## Proposed Approach
Three small, independently shippable changes — all in the sibling research repo plus minor archivist-prompt updates here. They sequence so that conventions exist before the archivist is asked to apply them, and the index protocol piggybacks on the archivist write path.

1. **Conventions doc + JD scaffold.** Add a short `README.md` at the research-repo root documenting the Johnny-Decimal area/category/ID scheme, the Obsidian frontmatter fields (`id`, `title`, `tags`, `captured`, `source_url`), and wiki-link rules (`[[JD-ID title]]`). Create the empty top-level JD area folders.
2. **Archivist updates: JD assignment + frontmatter + atomic INDEX.md.** Update the archivist agent prompt/workflow (introduced in Epic Issue 4) so that on each keep/expand decision it: assigns the next free JD ID in the chosen area, emits the frontmatter block, writes the article, and in the *same commit* appends/updates the corresponding `INDEX.md` entry (JD ID + title + 1-line description). Atomicity is achieved by performing both file writes before the commit.
3. **Stretch — citation freshness signal.** Extend the citation convention from `{url, captured}` to also include a freshness marker (`freshness: fresh|stale|unknown` derived from a configurable age threshold, default 12 months). If deferred, this Issue is closed with the deferral noted in the README.

The scope is convention/documentation plus small prompt edits — comfortably within one PG, but split into three Tasks so each AC bullet has a clear owner and the Stretch can be deferred cleanly without blocking the Minimum.
