# Plan — Document Invoke-PolyphonySdlc.ps1 and worktree convention in onboarding-guide.md

## Strategic Objective

Close the docs gap that forces new polyphony users to read source to discover (a) the canonical SDLC entry point [`scripts/Invoke-PolyphonySdlc.ps1`](scripts/Invoke-PolyphonySdlc.ps1) and (b) the sibling-worktree convention (`../polyphony-<slug>`) that `apex-driver@polyphony` assumes. Make [`docs/onboarding-guide.md`](docs/onboarding-guide.md) the on-ramp from fresh clone to first apex-driver invocation, with the [`polyphony-sdlc`](.github/skills/polyphony-sdlc) skill remaining the deep reference.

## Success Criteria

- Onboarding guide contains a section that walks a new user end-to-end from fresh clone to first `Invoke-PolyphonySdlc.ps1` invocation.
- Worktree convention is documented with a copy-pasteable recipe (`git worktree add`, `.twig` setup, `twig set`, `twig sync`).
- Wrapper parameter reference covers at minimum `-ApexId`, `-Intent`, `-Platform`, `-DryRun`, `-NoDetach` (matches the param block at [scripts/Invoke-PolyphonySdlc.ps1:68](scripts/Invoke-PolyphonySdlc.ps1)).
- At least one worked example shows the `-DryRun` output and how to interpret it.
- The `polyphony-sdlc` skill cross-links into the new onboarding section, so the on-ramp ↔ deep-reference split is explicit in both directions.
- No code changes; no rest-of-guide restructuring.

## Scope

### In Scope
- New section(s) in [`docs/onboarding-guide.md`](docs/onboarding-guide.md) covering the worktree convention and the `Invoke-PolyphonySdlc.ps1` wrapper (parameter reference + worked dry-run example).
- Cross-link addition in the `polyphony-sdlc` skill pointing to the new onboarding section.

### Out of Scope
- Code changes to [`scripts/Invoke-PolyphonySdlc.ps1`](scripts/Invoke-PolyphonySdlc.ps1) or anything else in the repo.
- Restructuring or rewriting the existing onboarding guide sections.
- Creating any new skill files (deep dives stay in `polyphony-sdlc`).

## Child Issues

- task-1 — Document the sibling-worktree convention in onboarding-guide.md — provides the bootstrap recipe a new user needs before they can run the wrapper.
- task-2 — Document `Invoke-PolyphonySdlc.ps1` wrapper (parameter reference + dry-run worked example) in onboarding-guide.md — the canonical first-invocation path; depends on the worktree section existing so it can link to it.
- task-3 — Cross-link from the `polyphony-sdlc` skill into the new onboarding section — closes the on-ramp ↔ deep-reference loop.

## Risks

- **Drift risk:** Wrapper params evolve; the doc must cite the param block by file path so future readers can verify against source.
- **Placement risk:** Onboarding guide already has 10 numbered sections. Inserting between §3 Bootstrap and §9 First Run risks renumbering; planner should prefer appending or slotting near §9 First Run rather than restructuring.
