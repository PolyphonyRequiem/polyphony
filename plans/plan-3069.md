# Wire cross-links between launcher gh-auth doc and neighbours

## Summary
Add the inbound and outbound cross-links required by the parent Epic's acceptance criteria so readers hitting a launcher gh-auth failure path can discover [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md), and so that doc points back to its provenance and the launcher script. This is a small, focused docs-only change implemented as a single PG.

## Problem / Motivation
The Epic ([#3067](#3067)) introduces a new troubleshooting doc covering the `gh` identity probe and competing-credential failure modes. A doc that no neighbour links to is effectively undiscoverable: readers hitting the failure mode in the launcher won't find it. Likewise, the doc needs outbound links so readers can pivot to the launcher script it documents and to the originating PR for provenance. This Issue closes the discoverability and provenance gap.

## Proposed Approach
Docs-only edits, no code changes:

1. **Inbound link** — add a contextual sentence (not a bare URL) linking to [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) from at least one sensible neighbour. Preferred target: [docs/onboarding-guide.md](docs/onboarding-guide.md) in the section that discusses prereqs / `gh` auth, since that's where a reader is most likely to hit the failure first. If a more natural anchor exists in [.github/skills/polyphony-bootstrap/SKILL.md](.github/skills/polyphony-bootstrap/SKILL.md) or [.github/skills/polyphony-sdlc/SKILL.md](.github/skills/polyphony-sdlc/SKILL.md), prefer the skill that already enumerates gh-related pitfalls.
2. **Outbound links** — inside [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md), add:
   - A link to [scripts/Invoke-PolyphonySdlc.ps1](scripts/Invoke-PolyphonySdlc.ps1) in a context that names it as the launcher being troubleshot.
   - A link to PR `PolyphonyRequiem/polyphony#250` (e.g., in a Provenance / History section at the bottom).

This Issue depends on the sibling Issue that creates [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) — that file must exist before the outbound edits and the inbound link can be made.

## Acceptance Criteria
- [ ] At least one neighbour doc/skill links to [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) with a contextual sentence (not just a bare URL).
- [ ] [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) contains an outbound link to [scripts/Invoke-PolyphonySdlc.ps1](scripts/Invoke-PolyphonySdlc.ps1).
- [ ] [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) contains an outbound link to PR `PolyphonyRequiem/polyphony#250`.
- [ ] Build passes with zero errors and warnings.
- [ ] All existing tests pass; no new tests required (docs-only change).

## Dependencies
- Sibling Issue under Epic [#3067](#3067) that creates [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) — target file must exist first.

## Context
This is a docs-only change well under the single-PG threshold (≤ ~2000 LoC, ≤ 50 files); decomposition into Tasks would add overhead with no benefit. Treated as an atomic, implementable unit of work.
