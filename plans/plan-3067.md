# Epic Plan — Add docs/troubleshooting/launcher-gh-auth.md

## Strategic Objective

Close the documentation gap left by PR #250 (the gh-identity probe in [scripts/Invoke-PolyphonySdlc.ps1](scripts/Invoke-PolyphonySdlc.ps1)) so users hitting wrong-account launches, competing-worker `gh auth switch` slippage, or probe failures have a single discoverable troubleshooting page that explains the new diagnostic line, the two failure modes the probe prevents, and a symptoms → remediation table for each error the probe surfaces.

## Success Criteria

- [docs/troubleshooting/launcher-gh-auth.md](docs/troubleshooting/launcher-gh-auth.md) exists and covers: probe trigger conditions (`-Platform github` AND not `-DryRun`), the `gh identity pinned: user='X' source=Y token_len=N` diagnostic line, and the two failure modes (competing-worker slippage; stale/wrong-scope token).
- The doc contains a symptoms → remediation entry, with verbatim actionable commands, for each error path surfaced by [scripts/Resolve-GhIdentity.ps1](scripts/Resolve-GhIdentity.ps1): "gh not on PATH", "no token cached" (incl. EMU-vs-PolyphonyRequiem account-switch case), "token validation failed" (401/403), "gh auth token timed out".
- At least one inbound cross-link from a neighbouring doc/skill (e.g. [.github/skills/polyphony-sdlc/SKILL.md](.github/skills/polyphony-sdlc/SKILL.md), [.github/skills/polyphony-bootstrap/SKILL.md](.github/skills/polyphony-bootstrap/SKILL.md), or [docs/onboarding-guide.md](docs/onboarding-guide.md)).
- At least one outbound cross-link back to [scripts/Invoke-PolyphonySdlc.ps1](scripts/Invoke-PolyphonySdlc.ps1) and PR #250 for provenance.
- All markdown links in the new doc resolve (no broken refs).
